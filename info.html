Frameworks

¿Que es un framework?

Un framework es un esquema o marco de trabajo que ofrece una estructura base para elaborar un proyecto con objetivos específicos, una especie de plantilla que sirve como punto de partida para la organización y desarrollo de software.

Pero bueno esa el la descripcion elegante con palabaras rimbombantes (llamativas) pero en palabras simples (mas simples), ¿que es un framework?, un Framework es una extension para realizar un trabajo de manera mas rapida y sencilla como por ejemplo react, uno de los Frameworks mas famosos, en este caso para desarrollo front end.

Historia y evolucion de los frameworks


Los frameworks de desarrollo de software han evolucionado significativamente a lo largo del tiempo. Comenzaron con bibliotecas de funciones compartidas en la década de 1970 y se expandieron con la programación orientada a objetos en la década de 1990. En la década de 2000, surgieron frameworks web que simplificaron el desarrollo de aplicaciones web. Hoy en día, los frameworks se centran en la modularidad, la programación funcional y la automatización, adaptándose a las necesidades cambiantes de la industria. La comunidad de código abierto desempeña un papel crucial en su desarrollo continuo, prometiendo un futuro emocionante para la creación de aplicaciones de alta calidad.

Tipos de frameworks


Django es un framework de desarrollo web de código abierto, escrito en Python, que respeta el patrón de diseño conocido como modelo–vista–controlador.
Spring es un framework para el desarrollo de aplicaciones y contenedor de inversión de control, de código abierto para la plataforma Java.​
REACT es una librería Javascript de código abierto diseñada para crear interfaces de usuario con el objetivo de facilitar el desarrollo de aplicaciones en una sola página.
Angular es un framework para aplicaciones web desarrollado en TypeScript, de código abierto, mantenido por Google.
Express es un entorno de trabajo para aplicaciones web para el programario Node.js, de código abierto y con licencia MIT.Se utiliza para desarrollar aplicaciones web y APIs.
VUE.JS es un framework de JavaScript de código abierto para la construcción de interfaces de usuario y aplicaciones de una sola página. Fue creado por Evan You, y es mantenido por él y por el resto de los miembros activos del equipo central que provienen de diversas empresas como Netlify y Netguru.



Ventajas y desventajas de los frameworks


Ventajas

  • Estructura y organizacion del codigo
  • Reutilizacion del codigo
  • Agilidad y rapidez en el desarrollo
  • Menor costo en el desarrollo
  • Buenas practicas de desarrollo
  • Minimizar errores y mayor facilidad para solucionarlos



Desventajas

  • Tiempo de prendizaje
  • Versiones inestables
  • Mayor consumo de recursos
  • Eleccion del framework




Manejo de peticiones web


HTTP define un conjunto de metodos de peticion para indicar la accion que se desea realizar para un recurso determinado. Aunque estos tambien pueden ser sustantivos, estos metodos de solicitud a veces son llamados HTTP verbs. Cada uno de ellos implementan una semantica diferente, pero algunas caracteristicas similares son compartidas por un grupo de ellos.



GET El método GET es adecuado para la personalización de páginas web: el usuario puede guardar búsquedas, configuraciones de filtros y ordenaciones de listas junto al URL como marcadores, de manera que en su próxima visita la página web se mostrará según sus preferencias.
POST El método POST introduce los parámetros en la solicitud HTTP para el servidor. Por ello, no quedan visibles para el usuario. Además, la capacidad del método POST es ilimitada.
HEAD El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta. El método POST se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
PUT se utiliza para sustituir totalmente un recurso de OSLC. El método PUT actualiza las propiedades de literal y las propiedades de recurso local y suprime las propiedades de recursos local que no estén incluidas en la solicitud.
DELETE El método DELETE borra un recurso en específico.




Tipos de web hosting


Para pensar que tipo de web hostinng necesita tu sitio, primero piensa que tan grande quieres que sea tu proyecto web y cuales son conociminetos tecnicos para enfrentarse a cuestiones como la administracion de un sitio web, a continuacion una lista de los tipos de web hosting.

Compartido En el hosting compartido, varios sitios web comparten los recursos de un mismo servidor. Es una opción económica y adecuada para sitios web pequeños o con tráfico moderado.
VPS Un servidor privado virtual es una opción intermedia entre el hosting compartido y el servidor dedicado. En un entorno de VPS, un servidor físico se divide en múltiples servidores virtuales, lo que proporciona más recursos y control que el hosting compartido.
Servidor Dedicado En el hosting con servidor dedicado, tienes un servidor completo para tu sitio web. Esto ofrece el máximo control y recursos, pero es la opción más costosa. Es adecuado para sitios web con alto tráfico y requisitos de rendimiento.
Cloud Hosting El hosting en la nube utiliza una red de servidores virtuales interconectados para alojar un sitio web. Esto permite escalabilidad instantánea y la capacidad de pagar solo por los recursos que se utilizan. Es adecuado para sitios web con necesidades de escalabilidad y puede ser rentable.
